# Баги

Пока изучал API, нашел достаточно много багов, такие как возможность ставить отрицательную цену или отрицательный sellerId и т.д.
Но тут описал только баг, который нашел с помощью авто-тестов.

### Некорректная валидация поля statistics при значении 0
**Краткое описание**
При создании объявления с полем statistics, где значения likes, viewCount или contacts равны 0, API возвращает ошибку 400 с сообщением, что поле обязательно. Согласно логике, 0 является допустимым значением (отсутствие лайков, просмотров и т.д.).

**Шаги воспроизведения**
1. Выполнить POST-запрос к эндпоинту /api/1/item
Передать в теле запроса:
```
{
  "sellerID": 864678,
  "name": "Телефон",
  "price": 90850,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}
```

**Фактический результат**
- Код ответа: 400 Bad Request
- Тело ответа:
```
{
  "result": {
    "message": "поле likes обязательно",
    "messages": {}
  },
  "status": "400"
}
```

**Ожидаемый результат**
- Код ответа: 200 OK
- Объявление создаётся со статистикой, где все поля равны 0

**Серьёзность**
Critical. Критическая ошибка, нарушающая основной функционал программы. Баг проявляется стабильно и блокирует использование ключевых функций.

**Окружение**
- URL: https://qa-internship.avito.com
- Эндпоинт: /api/1/item
- Заголовки: {"Content-Type": "application/json", "Accept": "application/json"}
- Тело запроса:
```
{
  "sellerID": 864678,
  "name": "Телефон",
  "price": 90850,
  "statistics": {
    "likes": 0,
    "viewCount": 0,
    "contacts": 0
  }
}
```
